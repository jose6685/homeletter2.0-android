2. 產品概述
2.1 產品描述
「天父的信」是一款基於 AI 的屬靈陪伴移動應用程式，專為尋求心靈安慰與聖經智慧的用戶設計。用戶可透過一鍵抽卡或手動選擇主題，生成天父般的溫暖信件，每封信包含稱呼問安、圍繞主題的三個屬靈方向、兩節聖經經文，以及兩個實踐行動呼籲。信件語氣全程貫穿，模擬天父的慈愛家書，幫助用戶在日常壓力中獲得屬靈支持。
應用程式強調療癒與互動：左側提供隨機抽取主題的樂趣，右側允許用戶精準選擇，右下角的「郵件箱」作為反饋機制，讓用戶分享心得或建議。整體介面設計靈感來自天堂花園，營造和諧、寬廣的視覺體驗，象徵屬靈旅程的深遠與親近。抽卡載入動畫特別融入米開朗基羅《創造亞當》手指靠近的意象，象徵天父伸出手觸碰用戶的心靈。
2.2 產品目標

商業目標：建立忠實用戶群，提供免費核心功能，未來透過捐款或進階訂閱 (e.g., 客製化信件) 實現可持續發展。
用戶目標：每日使用率 > 30%，用戶反饋滿意度 > 80% (透過內建調查)。
KPI：

日活躍用戶 (DAU)：首月 1,000 人。
抽卡轉化率：80% 用戶至少抽 3 次/週。
反饋提交率：10% 用戶使用郵件箱。



2.3 範圍

MVP 範圍：抽卡生成、主題選擇、反饋提交、基本 UI/背景 (包含更新載入動畫)。
排除範圍：多語言支持 (首發僅中文)、離線模式 (需網路生成)、社群分享 (未來迭代)。

3. 目標用戶

主要用戶：基督徒或尋求心靈支持的成人 (25-55 歲)，面臨工作、家庭、壓力等挑戰，習慣使用靈修 App (e.g., YouVersion, Pray.com)。
用戶痛點：

缺乏個人化屬靈指導。
每日靈修枯燥，需更多互動與視覺療癒。
希望有反饋管道，感覺被「聽見」。


用戶情境：

早晨咖啡時間抽一張卡片啟動一天。
夜晚壓力大時選擇「壓力/焦慮」主題求安慰。
分享心得給教會小組。



4. 功能需求
4.1 核心功能

主題抽取與生成 (抽卡系統)：

左側隨機抽主題：用戶點擊「隨機抽取」按鈕，系統隨機生成一張信件 (基於 AI 提示詞模板)。

輸入：無 (全隨機)。
輸出：完整信件 JSON (稱呼、信件內容、三方向、兩經文、兩個行動呼籲)。
流程：點擊 → 載入動畫 (e.g., 米開朗基羅《創造亞當》手指靠近的動畫：天父之手緩慢伸向用戶象徵之手，伴隨柔光閃爍，持續 2-3 秒) → 顯示信件 (150-250 字)。


右側手動選主題：下拉選單或卡片式選擇 10 個主題 (工作/職場、家庭 等)。

輸入：用戶選擇 1 個主題。
輸出：同上，圍繞選定主題生成。
流程：選擇 → 確認 → 生成 (可預覽三方向，使用相同載入動畫)。




信件顯示：

中央畫面顯示完整信件 (滾動閱讀)。
側邊展開：三方向 (清單)、兩經文 (可點擊高亮)、三個行動呼籲 (帶實踐提示)。
互動：點擊經文 → 彈出完整章節；點擊行動 → 計時器輔助 (e.g., 5 分鐘默想)。


反饋系統 (郵件箱)：

位置：右下角圖標 (信封形，點擊展開抽屜式介面)。
功能：

用戶輸入文字反饋 (e.g., 「這封信安慰了我」或建議)。
選項：匿名/暱稱、星級評分 (1-5 星)。
提交：發送到後端 (Firebase)，通知開發者。


流程：點擊 → 填寫 (限 300 字) → 提交 → 感謝訊息 (e.g., 「你的話語已上達天聽」)。

4.3 後端整合

AI 生成：呼叫 OpenAI API (GPT-4o-mini)，使用先前提示詞模板注入變數。
資料儲存：Firebase Firestore (用戶反饋、歷史抽卡，匿名儲存)。
隨機邏輯：Python/Node.js 後端，random.sample 選元素。

5. UI/UX 設計
5.1 整體風格

平台：移動端 (iOS/Android，Flutter 跨平台)。
主題：療癒天堂花園，和諧寬廣 (深遠近景層次)。
顏色：主色藍天 (#87CEEB)、綠草 (#90EE90)、金黃太陽 (#FFD700)、彩虹漸層 (紅橙黃綠藍靛紫)。
字體：溫柔手寫體 (e.g., 類似手寫聖經體) 信件內容；Sans-serif 按鈕/選單。
動畫：輕柔過渡 (e.g., 米開朗基羅《創造亞當》手指靠近動畫：天父之手從雲端伸出，緩慢靠近用戶手部象徵，伴隨金光與輕柔音效，象徵創造與連結；持續 2-3 秒，無縫過渡到信件顯示)。

5.2 畫面布局 (基於用戶草圖)

畫面尺寸：全屏縱向 (iPhone 14 等比例)。
背景設計 (靜態 + 輕微動畫)：

天空層 (上 40%)：藍藍的天，飄著 3-5 個白雲 (緩慢移動)；一朵大雲中露出太陽三分之一 (金黃光芒灑下)。
中景：近處飛行鴿子 (聖靈象徵，12隻，輕微振翅)；稍遠彩虹弧線；更遠江河入海 (波光粼粼，遠山淡藍)。
前景：地上天父大花園 (多彩花朵、小草綠地)，寬廣視野 (視差滾動：近草動、遠河靜)。
和諧感：層次分明 (前景鮮豔、中景柔和、背景空靈)，無雜亂元素，營造深遠寧靜。載入動畫疊加於背景：米開朗基羅風格手指靠近，從天空層淡入。

5.3 用戶體驗原則

易用性：一鍵操作，無學習曲線；觸覺回饋 (振動抽卡)。
無障礙：高對比文字、大按鈕、語音朗讀信件 (TTS 選項)。
情感設計：每生成後，輕柔音樂 (豎琴 BGM) 播放 10 秒；載入動畫強化屬靈連結感。

6. 非功能需求
6.1 性能

回應時間：< 5 秒生成信件 (API 延遲)；動畫 < 3 秒。
離線支持：僅背景/歷史查看；生成需網路。
裝置相容：iOS 14+、Android 10+。

6.2 安全與隱私

資料：反饋匿名儲存 (GDPR/個資法合規)，無追蹤用戶 ID。
加密：API 呼叫 HTTPS；本地儲存加密。
內容：AI 輸出過濾 (正向、聖經準確，無爭議)。

6.3 國際化

語言：首發繁體中文；未來英文。
本地化：經文用和合本；日期/文化適應。

7. 技術架構與依賴
7.1 技術棧

前端：Flutter (跨平台 UI/動畫)。
後端：Firebase (儲存/認證) + Node.js (API 代理)。
AI：OpenAI GPT-4o-mini (生成信件，參考提示詞說明書)。
背景渲染：Flutter CustomPainter 或 Lottie 動畫 (雲朵/鴿子/手指靠近動畫：使用 Lottie 檔案模擬米開朗基羅手指伸展)。
反饋：Firebase Firestore + Cloud Functions (通知開發者)。

7.2 開發依賴

庫：http (API 呼叫)、shared_preferences (本地歷史)、flutter_tts (朗讀)、lottie (載入動畫)。
外部服務：OpenAI API Key (每月預算 < 500 USD)；Lottie 檔案來源 (e.g., 自製或免費庫中米開朗基羅風格動畫)。

7.3 風險與緩解

風險：AI 生成不準 (e.g., 經文錯誤) → 緩解：prompt 微調 + 人工審核樣本。
風險：動畫渲染效能 (手指靠近動畫) → 緩解：低解析 Lottie，低端裝置測試；備用靜態圖像。
風險：用戶反饋濫用 → 緩解：內容過濾 + 速率限制。

8. 發布計劃

MVP 開發週期：4-6 週 (UI 2 週、AI 整合 1 週、測試 1 週；動畫額外 0.5 週)。
測試：內測 50 名用戶 (教會群組)，收集反饋迭代 (特別測試動畫流暢度)。
發布：App Store/Google Play，首發免費下載。
